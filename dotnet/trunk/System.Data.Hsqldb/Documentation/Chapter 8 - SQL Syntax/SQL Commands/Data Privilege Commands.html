Chapter 8 - SQL Syntax
Chapter Overview > SQL Commands > Data Privilege Comands

GRANT

GRANT { SELECT | DELETE | INSERT | UPDATE | ALL } [,...]
   ON { table | CLASS "Fully.Qualified.Type.Name" }
   TO <grantee>;

GRANT <rolename> [,...]
   TO <grantee>;

Requires Administrative privileges.

<grantee> is either a user name, a role name, or PUBLIC, which effectively means all users.

The first form of the GRANT command assigns privileges to a grantee for table access or method invocation upon a type.

To allow a user to invoke a method of a type from SQL, the privilege ALL must be granted. Examples:

    GRANT SELECT ON Test TO GUEST;
    GRANT ALL ON CLASS "System.String" TO PUBLIC;

The second form of the GRANT command gives the specified <grantee> membership in the specified role.

REVOKE

REVOKE { SELECT | DELETE | INSERT | UPDATE | ALL } [,...]
    ON { table | CLASS "Fully.Qualified.Type.Name" }
  FROM <grantee>;

REVOKE <rolename> [,...]
  FROM <grantee>;

Requires Administrative privileges.

<grantee> is either a user name, a role name, or PUBLIC.

The first form of the REVOKE command withdraws privileges from a grantee on a table or for invocation of methods on a type.

The second form of the REVOKE command withdraws membership of the specified <grantee> from the specified role.

Note

There are a number of special (predefined) roles and users:

  • _SYSTEM - the user that grants predefined privileges to other predefined users/roles.
  • PUBLIC -a (virtual) user that automatically grants its privileges to all normal roles and users
  • DBA - the database administrator role.

    • Possessing this role, either directly or indirectly, grants unrestricted access to all features of the database instance.
    • Upon creation, each new database instance contains a single user "SA" with an empty password and membership in the DBA role.
    • Users with the DBA role may use the ADMIN option of the CREATE USER command to grant the DBA role as part of the user creation operation.

It is illegal to attempt to alter the privileges of _SYSTEM or DBA.

It is not possible to connect as _SYSTEM or PUBLIC.
